Guidelines for scheduling jobs

This page provides an overview of scheduled Jobs in IE-MDM.

A job is a scheduled IE-MDM task. It is possible to change the schedule for a job but also to trigger a job manually. For the scheduling timing and interval a Cron Expression is used. Only valid Cron Expressions are accepted. To get more information about Cron Expressions, and to check their validity, an external site can be used such as FreeFormatter.

Jobs can be paused. While paused, the job will not be executed according to its schedule, but the configured schedule will stay in place. When a job is resumed, the job will be executed again according to the defined schedule. The jobs are also non-reentrant, meaning that the same job cannot be executed while still running. If a scheduled job cannot start because a current execution is still running, then it will automatically be rescheduled to start when the current execution is finished. If a manually triggered job cannot start it will not be rescheduled.
